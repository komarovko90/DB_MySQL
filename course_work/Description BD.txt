Курсовая работа. 

Проектирование и создание тестовой БД с использованием СУБД MySQL для каршерингового сервиса ЯндексДрайв. 

Описание БД.

Таблицы БД:
users
profiles
cars
status_price
session
routes
damages
damages_photo
rental
car_services
car_repair

Users
Таблица для хранения основных данных пользователя, необходимые для использования мобильного 
приложения: авторизации (почта, телефон), оплаты(карта). 

Profiles
Таблица для хранения дополнительной информации о пользователе - паспортные данные, в/у и тд.

Cars
Таблица для хранения информации об арендуемых автомобилях.

Class_price
Таблица для хранения информации о классификации авто и их соответствующей стоимости аренды.

Rental
Таблица для хранения информации об аренде автомобиля пользователем. 
В таблицах хранятся основные сведения об аренде: авто, кто арендовал.

Routes
Таблица для хранения информации о проделанном маршруте при аренде авто. 

Damages_car
Таблица для хранения информации о повреждениях авто, которые 
регистриуются пользователем в приложении при бронировании авто. 

Damages_photo
Таблица для хранения ссылок на фотографии, которые производятся 
пользователем при бронировании авто. 

Rental_time
Таблица для хранения информации о нахождении авто в разных режимах аренды: ожидание, поездка. 
Пользователь может в любое время переводить авто из режима поездки в режим ожидания и, наоборот. 
Стоимость данных режимов различается. 

Car_services
Таблица для хранения информации о сервисах, производящих ремонт и обслуживание авто.

Car_repair
Таблица для хранения информации о произведенных ремонтах и тех. обсулуживаниях авто.
